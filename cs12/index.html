Python Revision Tour

Revisiting the essential structural pillars of Python. This review establishes key variables, lexical rules, sequence operations, and native mapping strategies.

Tokens and Identifiers

A token represents the smallest individual syntax unit that the Python interpreter processes. Python recognizes five distinct kinds of tokens:

  • Keywords: Reserved words that have preset semantic definitions within the compiler namespace (e.g. False, class, def, lambda, nonlocal).
  • Identifiers: Unique labels assigned to variables, objects, and user-defined helper methods.
    • Must begin with an alphabetical character [a-zA-Z] or a leading underscore _.
    • Can contain numeric values but cannot start with a digit.
    • Special character symbols (e.g., $, @) and space characters are strictly prohibited.
    • Keywords cannot be used as identifier labels.
  • Literals: Values that remain constant:
    • String: Defined inside single, double, or triple quotation marks.
    • Numeric: Signed integers (decimal, octal e.g. 0o12, hex e.g. 0xAF), float values (decimal and scientific e.g. 1.4e-3), and complex numbers (e.g. 2 + 3.5j).
    • Boolean: True and False.
    • Special: The None value.

Data Types & Type Conversion

Python variables map values dynamically. Standard built-in data types include:

value_int = 101 value_float = 28.75 value_complex = 3 + 4j value_str = "APEX" value_bool = True

Type conversion is divided into:

  • Implicit Conversion: The interpreter automatically changes data types to prevent data loss (e.g. dividing an integer by float automatically yields a float representation).
  • Explicit Conversion (Casting): Done manually by calling type conversion methods like int(), float(), str(), bool(), list(), tuple().

Operators in Python

Operators process variables and literals. Python categorizes them as follows:

  • Arithmetic: Modulo % (returns division remainder), Floor division // (rounds division result down to the nearest integer), exponentiation **, addition, subtraction, multiplication, and division.
  • Relational: Compare values (==, !=, <, >, <=, >=), returning boolean outputs.
  • Logical: Chains conditional evaluations using and, or, and not.
  • Assignment: Assigns values (=) and modifies them inline (e.g., +=, *=, //=).
  • Identity: is and is not check memory location identities (verifying if id(a) == id(b), unlike value checks like a == b).
  • Membership: in and not in check whether a value exists inside sequences (like lists, strings, or tuple elements).

Control Structures

Control structures manage the flow of code execution:

  • Conditional Branching: The if-elif-else statement evaluates condition values to choose which code block runs.
  • Looping Iterations:
    • while: Loops as long as the condition holds.
    • for: Cycles through items in a sequence or a numeric range().
    • Both loops support an optional else block, which runs when the loop finishes normally without hitting a break.
  • Control Words: break (abruptly exits the loop), continue (skips the rest of the current iteration, returning to the loop header), and pass (acts as a syntactic placeholder statement).

Strings, Lists, and Tuples

Python organizes ordered data collections as sequences:

  • Strings: Immutable Unicode sequences. Operations include:
    • Indexing: Access characters using positive (0 to len-1) or negative (-1 to -len) offsets.
    • Slicing: Retrieve substrings using [start:stop:step].
    • Methods: upper(), lower(), find(), split(), strip(), replace().
  • Lists: Mutable, ordered collections. Elements can be updated in-place. Methods include append(), insert(), remove(), pop(), sort(), and extend(). List comprehensions offer syntax for inline creation: [x**2 for x in values if x > 0].
  • Tuples: Immutable sequences. Once created, their elements cannot be changed. Created using parentheses (). Single element tuples require a trailing comma: (5,).

Dictionaries

Dictionaries represent mapping structures that match key objects to data values:

  • Keys must be immutable types (e.g. strings, integers, tuples). Duplicate keys are not allowed.
  • Values can be accessed using dict[key] or the safe lookup method dict.get(key, default), which avoids raising a KeyError.
  • Core operations: keys() (retrieves all keys), values() (retrieves all values), items() (retrieves key-value tuple pairs), and update() (merges another dictionary).

Functions

Structure your programs by dividing code logic into modular, reusable blocks of execution.

Need for Functions

Boilerplate code is hard to read and debug. Functions partition programs, providing:

  • Modularity: Divide a complex project into smaller, manageable, and readable sub-tasks.
  • Reusability: Define execution logic once, then call it from multiple modules without duplicating code.
  • Isolated Namespaces: Localize variables, preventing naming conflicts across modules.

Types of Functions

  • Built-in: Functions available natively in the language runtime namespace (e.g. len(), range(), print()).
  • Module-based: Stored inside standard library files, requiring imports (e.g., math.sqrt(), random.randint()).
  • User-defined: Created by developers using the def keyword.

Arguments and Parameter Rules

Parameters act as formal placeholders in a function definition, while arguments are the actual values passed during a function call.

  • Positional Arguments: Arguments map to parameters in order, from left to right.
  • Default Parameters: Pre-assign fallback values to parameters. Rule: Default parameters must be declared after all non-default parameters. E.g., def db_connect(host, port=3306) is legal, but def db_connect(port=3306, host) is illegal.
  • Keyword Arguments: Let callers pass values using parameter names, allowing them to change the order. E.g., db_connect(port=3306, host="127.0.0.1").
  • Variable-length Arguments:
    • *args: Packs extra positional arguments into a tuple.
    • **kwargs: Packs extra keyword arguments into a dictionary.

Variable Scoping Rules

Scope defines the accessibility of variables inside different regions of a program:

  • Local Scope: Created inside a function body. Accessible only within that function and destroyed on return.
  • Global Scope: Variables declared outside all function blocks. Accessible throughout the module.
  • The global Keyword: Declared inside a function to allow modifying a variable in the global scope. E.g., global counter.
  • Lifetime: The period of time a variable occupies memory. Local variables exist only during function execution, whereas global variables persist until the script finishes.

Lambda Functions

Lambda functions are small, anonymous, single-expression functions: lambda arg1, arg2: expression. They are useful as quick callbacks or inputs to functions like map() or filter().

Recursion Concepts

Recursion is a programming technique where a function solves a problem by calling itself with modified inputs.

  • Base Case: The termination condition that returns a value directly without making further recursive calls, preventing infinite loops.
  • Recursive Case: The step where the function calls itself with modified parameters, moving closer to the base case.
def fibonacci(n): if n <= 1: return n return fibonacci(n-1) + fibonacci(n-2)

Exception Handling

Catch runtime errors to build resilient, crash-proof programs.

Errors vs Exceptions

  • Syntax Errors: Violations of Python's language rules, caught by the parser before the script runs. Prevents execution (e.g. missing colon).
  • Runtime Errors (Exceptions): Errors detected during execution, interrupting normal program flow (e.g., dividing by zero, passing invalid types).
  • Logical Errors: Semantic bugs where the program runs without crashing but produces incorrect results.

Need for Exception Handling

Uncaught exceptions terminate programs, causing crashes. Exception handling allows a program to catch these errors and recover gracefully, log diagnostics, or clean up system resources.

The Control Flow

Python manages exceptions using four structured blocks:

  • try: Contains the block of code to be monitored for exceptions.
  • except: Contains exception handler logic. Matches the type of exception raised in the try block. Multiple except blocks can be chained to handle different exception types.
  • else: Runs only if the code in the try block executes successfully without raising any exceptions.
  • finally: Always runs, regardless of whether an exception was raised or handled. Useful for cleanup actions like closing database connections or files.

Common Built-ins & Custom Types

  • ZeroDivisionError: Division or modulo operation with a zero denominator.
  • ValueError: Raised when a function receives an argument of the correct type but an inappropriate value (e.g. int("abc")).
  • TypeError: Mismatched operation types (e.g. adding a string to an integer).
  • IndexError: Subscript index out of range for a sequence.
  • KeyError: Key not found in a dictionary.
  • NameError: Accessing an undefined variable name.
  • FileNotFoundError: Trying to open a file that does not exist.
  • The raise statement: Manually triggers an exception. E.g., raise ValueError("Limit exceeded").
  • User-defined Exceptions: Custom exceptions created by subclassing the base Exception class: class CustomError(Exception): pass.

File Handling

Save and read persistent data files in text, binary, and CSV formats.

Text Files

Text files store characters line by line, terminated by an EOL (End of Line) character. Python uses the built-in open() function to open them.

  • Modes:
    • r: Read-only (default). File must exist.
    • w: Write-only. Overwrites if file exists, creates it if not.
    • a: Append. Writes data to the end of the file.
    • r+ / w+ / a+: Open file for both reading and writing.
  • Reading: read(n) (reads n bytes/chars), readline() (reads one line), readlines() (returns a list of all lines).
  • Writing: write(string) (writes string data), writelines(list) (writes a list of strings).
  • Context Manager: The with statement automatically closes files upon exiting the block, ensuring safe handle release.

Binary Files

Binary files store raw bytes, matching the representation of data in memory. Opened using modes rb, wb, and ab.

  • The pickle Module: Used to serialize (convert python objects into byte streams) and deserialize data.
  • pickle.dump(obj, file): Serializes and writes data structures to a binary file.
  • pickle.load(file): Reads and deserializes data, reconstructing the original python object.
  • Why Binary Mode? Preserves complex data types (like nested lists, dictionaries, or custom classes) without converting them to text strings.

CSV Files

Comma-Separated Values are managed using Python's built-in csv module, designed to parse structured row-based records.

  • csv.reader(file): Returns an iterator that parses each line of the CSV file into a list of strings.
  • csv.writer(file): Returns a writer object that writes list data into comma-separated rows.
  • csv.DictReader(file): Maps the columns in each row to a dictionary, using the headers in the first line as keys.
  • csv.DictWriter(file): Writes dictionaries to rows, matching fields against a predefined list of column headers.
  • Delimiters: Custom delimiters can be specified during reader/writer creation: csv.writer(f, delimiter=';').

File Pointer Manipulation

Python tracks the current position in a file using a cursor called the file pointer.

  • tell(): Returns an integer indicating the current position of the file pointer (in bytes from the start).
  • seek(offset, whence): Moves the file pointer by offset bytes relative to whence:
    • whence = 0: Offset calculated relative to the start of the file.
    • whence = 1: Offset calculated relative to the current cursor position.
    • whence = 2: Offset calculated relative to the end of the file.

Data Structures

Implement stacks and queues using standard Python list operations.

Stack Data Structure

A stack is a linear collection of elements that follows the LIFO (Last In First Out) principle. The last element added is the first one removed.

  • Push: Inserts an element onto the top of the stack.
  • Pop: Removes and returns the top element. Trying to pop from an empty stack causes an Underflow error.
  • Peek/Top: Returns the value of the top element without removing it.
  • Applications: Expression parser evaluation, undo/redo buffers, and search backtracking (DFS).

Queue Data Structure

A queue is a linear collection that follows the FIFO (First In First Out) principle. The first element added is the first one removed.

  • Enqueue: Inserts an element at the rear of the queue.
  • Dequeue: Removes and returns the element at the front.
  • Applications: Print buffers, cpu task dispatching queues, and graph traversals (BFS).

List-Based Implementation

We can simulate stacks and queues using standard Python lists:

stack = [] stack.append(10) val = stack.pop() queue = [] queue.append(20) val = queue.pop(0)

Computer Networks

A comprehensive overview of network scales, physical topologies, active protocols, devices, security threats, and regulations.

Types of Networks

Networks scale by geographic coverage, defining unique performance ranges:

  • PAN (Personal Area Network): Interconnects devices close to a person (range: 1-10 meters). E.g. Bluetooth mouse, wireless headphones.
  • LAN (Local Area Network): Covers small physical locations like offices, schools, or homes. Typically operates via Ethernet or Wi-Fi.
  • MAN (Metropolitan Area Network): Covers larger geographic areas like towns or cities (e.g. cable TV networks).
  • WAN (Wide Area Network): Links distant regions across countries or continents. The Internet is the largest global WAN.

Network Topologies

Topologies define the layout of links and nodes in a network:

  • Bus: Nodes share a single central backbone cable. If the backbone breaks, the entire network fails.
  • Star: Nodes connect to a central hub or switch. Highly reliable (one node failure doesn't affect others), but vulnerable to central switch failures.
  • Ring: Nodes form a circular loop, passing data tokens sequentially. A single node failure interrupts the whole ring.
  • Mesh: Nodes connect directly to each other. Highly redundant, reliable, and secure, but complex and expensive to configure.
  • Tree: A hierarchical combination of star and bus layouts, common in structured corporate LAN networks.

Hardware Devices

  • Modem (Modulator-Demodulator): Converts digital signals from a computer to analog signals for telephone lines, and vice versa.
  • Hub: A basic Layer-1 device that broadcasts incoming packets to all physical ports.
  • Switch: A Layer-2 device that reads MAC addresses to send packets only to their target destination, saving bandwidth.
  • Repeater: Amplifies signals to prevent packet loss over long physical distances.
  • Router: A Layer-3 device that forwards packets across different networks using IP routing tables.
  • Gateway: Connects networks that use different underlying protocols or architectures.

Transmission Media

  • Wired Media: Twisted Pair (STP/UTP), Coaxial cables (TV feeds), and Fiber Optic cables (transmits light signals through glass threads, immune to electromagnetic interference).
  • Wireless Media: Radio waves, microwaves, satellites (global line-of-sight), and infrared (short-range line-of-sight, e.g. remote controls).

Communication Protocols

  • TCP/IP: The core protocol suite of the Internet. TCP handles packet construction, sizing, and reliability, while IP manages addressing.
  • HTTP / HTTPS: Delivers web resources. HTTPS is encrypted using SSL/TLS protocols.
  • FTP: Used to transfer files between client and server systems.
  • PPP (Point-to-Point Protocol): Establishes a direct connection between two nodes (e.g. modem to ISP).
  • SMTP: Sends email messages, while POP3 handles downloading emails from mail servers.

Network Security & Cyber Law

  • Firewall: Monitors and filters incoming network packets based on predefined security rules.
  • Malware: Malicious software, including:
    • Viruses: Attach to host files and run when the host is executed.
    • Worms: Self-replicating programs that spread across networks, consuming bandwidth.
    • Trojan Horses: Disguised as safe files, they create backdoors for unauthorized access.
    • Spam: Unsolicited bulk email messages.
  • Cyber Crime & Law: The Indian IT Act 2000 provides the legal framework for cyber offenses like phishing (identity theft), hacking, and software piracy.
  • Web Services: Includes IoT (Internet of Things) and cloud computing (SaaS, PaaS, IaaS).
